Neubrutalist Components

How to Use

Include the base styles from https://joshforisha.github.io/neu-components/styles.css. This file outlines all of the CSS custom variables used by the components, and enables you to override any values you'd like in your own CSS.

You can then include the listed JS file for a component, and use the custom element as outlined. Enjoy!

Colors

Any component with a color attribute accepts any of the following eight values. Defaults are gray or neutral, and you can override any color as a custom variable in :root.

--red --yellow --green --teal --cyan --blue --purple --pink

There are also three neutrals and five tint/shade utility values:

--black --gray --white --darker --dark --light --lighter --lightest

<neu-block>

A block has a "floating" style and can act as a link when given an href value.

Attributes

The content of the block component is rendered below the heading element.

  • color: determines the color theme (default: gray/white)
  • external: (with a href) opens a new tab/window when clicked
  • heading: text title for the heading
  • href: URL that opens when clicked

<neu-button>

Green button

Attributes

  • color: determines the color theme
  • disabled: disables the button
  • external: (with a href) opens a new tab/window when clicked
  • href: target URL to open (acts as a link)
  • round: rounds the button to an ellipse/"pill" shape

<neu-card>

A card is flat, and typically used for grouping information.

Attributes

  • color: determines the color theme (default "white")
  • heading: text title for the heading

<neu-checkbox>

Attributes

  • checked: sets the state to checked
  • color: themes the check indicator
  • disabled: disables the input
  • label: labels the control

<neu-input>

Attributes

  • autofocus: automatically focus the input field
  • disabled: disables the input
  • leading: labels the control in the lead position
  • placeholder: rounds the button to an ellipse/"pill" shape
  • trailing: labels the control in the trail position

<neu-number>

Attributes

  • autofocus: automatically focus the input field
  • disabled: disables the input
  • leading: labels the control in the lead position
  • max: minimum value for a number control
  • min: minimum value for a number control
  • placeholder: rounds the button to an ellipse/"pill" shape
  • small: renders a shorter input
  • step: incremental value for a number control
  • trailing: labels the control in the trail position

<neu-radio>

Attributes

  • color: themes the selected indicator
  • disabled: disables the input
  • label: labels the control

<neu-select>

A <neu-select> is set up and used very much like a normal <select>: you include <option> elements inside.

Attributes

  • disabled: disables the input
  • leading: labels the control in the lead position
  • trailing: labels the control in the trail position

<neu-slider>

Attributes

  • disabled: disables the input
  • leading: labels the control in the lead position
  • max: maximum value
  • min: minimum value
  • step: incremental value
  • trailing: labels the control in the trail position

<neu-textarea>

Attributes

  • disabled: disables the input
  • leading: labels the control in the lead position
  • placeholder: sets the underlying field's empty placeholder text
  • trailing: labels the control in the trail position

<neu-toggle>

Attributes

  • color: themes the enabled state
  • disabled: disables the toggle
  • label: labels the control
  • toggled: sets the toggled state
